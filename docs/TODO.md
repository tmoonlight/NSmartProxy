# NSmartProxy TODO

按风险和收益排序，先稳定基础能力，再补充高级功能。

## P0：安全与稳定性

- 将单连接 1 GiB 缓冲区改为 32–64 KiB 池化缓冲区。
- 为所有协议帧增加长度上限，并使用完整读取，防止异常分配和短读。
- 管理端与控制链路启用 TLS；移除默认 `admin/admin`，默认关闭匿名登录。
- 使用标准密码哈希、短期令牌和角色权限；Debug/Release 保持相同鉴权。
- 统一连接生命周期、取消和超时处理，修复未回收任务、连接及传输状态。
- 客户端重连改为带抖动的指数退避。

## P1：可运维性

- 设计带版本协商的协议 v2，并保留旧协议兼容模式。
- 增加有界工作连接池、TCP 流复用、健康检查和后端负载均衡。
- 增加配置校验、端口白名单、客户端配额、连接与带宽限制。
- 提供结构化日志及代理级连接、流量、错误和重连指标。

## P2：扩展能力

- 按需求增加 QUIC/WSS、P2P、端口复用和插件机制。
- 统一受支持的 .NET 目标框架，修正跨平台发布命名和 RID。
- 建立单元测试、协议畸形包测试、并发压力测试及端到端 CI。

## 验收基线

- 多连接压力下内存稳定，无超大数组和未回收连接。
- 未认证用户无法访问管理 API，默认部署不会暴露弱口令。
- 断网、服务重启及配置更新后能够自动恢复代理。
- 每次提交均通过构建、单元测试和端到端测试。
